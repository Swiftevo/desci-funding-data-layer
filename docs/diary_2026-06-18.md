## 8. Donation Preservation Decisions

Confirmed:

```text
All donation rows should be preserved.
status = blank rows should be preserved.
success = FALSE rows should be preserved.
status, success, rawScore, threshold, coefficient, type, and last_score_timestamp must be preserved.
```

Reason:

```text
If failed or blank-status rows are removed or flattened, future AI agents may incorrectly treat all donation events as successful donations.
Preserving status and success fields is required for QF, threshold, donor graph, and anti-sybil analysis.
```

## 10. Cross-Round Identity Issue

Important confirmed issue:

```text
Some GG23 projects may also have participated in Spark S6.
The same real project may therefore currently have a Spark project_entity_id and a separate GG23 source project/application identity.
```

Decision:

```text
Do not immediately assign new permanent DSPJ IDs to GG23 projects.
Do not directly merge GG23 into canonical projects_latest before matching.
```

Correct workflow:

```text
1. Generate normalized GG23 draft import
2. Generate Spark/GG23 match candidates
3. Human confirms matches
4. Reuse existing Spark project_entity_id for confirmed matches
5. Assign new DSPJ IDs only to confirmed new projects
```

Reason:

```text
This avoids duplicate permanent project IDs for the same real DeSci project.
```
